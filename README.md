#Информационная система «Страховая компания»
Данный репозиторий содержит проект комплексной автоматизации деятельности страховой компании. Целью разработки является создание информационной системы для управления ключевыми процессами: оформление и продление страховых полисов, учет платежей, обработка заявлений о страховых случаях, выплата возмещений, а также управление персоналом и финансовая отчетность.

#Архитектура и моделирование
Проект выполнен с использованием структурного (SADT) и объектно-ориентированного подходов, что позволяет рассмотреть систему с различных точек зрения: от постановки задачи до физической реализации базы данных.

#В документе имеются следующие диаграммы для описания проекта:

Модели прецедентов (UML): Диаграммы, описывающие функциональные требования и взаимодействие актеров (клиент, страховой агент, бухгалтер, директор) с системой.

Диаграммы потоков данных (DFD): Детальное отображение движения информационных потоков между процессами, внешними сущностями и хранилищами данных (контекстный уровень и декомпозиция подсистем).

Функциональные модели (IDEF0): Иерархическое описание бизнес-процессов компании с указанием входов, выходов, управляющих воздействий и механизмов исполнения.

Модели потоков работ (IDEF3): Сценарии выполнения ключевого прецедента «Оплата страхового полиса» (PFDD) и диаграмма переходов состояний объекта (OSTN).

Модели данных (IDEF1X): Логическая и физическая модели базы данных, включающие сущности «Клиент», «Сотрудник», «Полисы», «Страховки», «Платежи», «Заявления» и связи между ними.

Метрики ПО: Расчет размерно-ориентированных (LOC) и функционально-ориентированных (FP) метрик для оценки трудоемкости, качества и документированности проекта.

#Назначение
Проектная документация предназначена для формирования целостного представления о будущей системе и может служить основой для непосредственной разработки программного обеспечения. 
#Решение позволяет:

Централизовать хранение данных о клиентах, договорах и финансах.

Сократить время обработки заявок и минимизировать ошибки ручного ввода.

Обеспечить прозрачность бизнес-процессов и контроль деятельности сотрудников.
